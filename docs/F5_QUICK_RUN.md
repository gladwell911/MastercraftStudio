# F5 快速运行

F5 快速运行面向桌面客户端里的代码助手工作流，作用类似轻量版 `Ctrl+F5`：先选项目，再直接运行项目主程序。

## 当前行为

- `Ctrl+O`：选择项目目录
- `F5`：在已加载项目上快速运行主程序
- 仅在 `codex` 或 `claudecode` 模型工作流下启用

## 运行规则

- 主程序文件按以下优先级查找：`main.py`、`app.py`、`run.py`、`__main__.py`、`start.py`、`index.py`
- Python 解释器优先使用项目虚拟环境：`.venv`、`venv`、`.venv311`、`env`
- 控制台程序会捕获输出
- GUI 程序会以独立窗口方式启动

## 使用步骤

1. 按 `Ctrl+O` 选择项目目录。
2. 确认当前模型是 `codex` 或 `claudecode`。
3. 按 `F5` 运行主程序。
4. 在状态栏和控制台查看结果。

## 适合保留的认知

- 这是一个开发辅助功能，不是通用进程管理器。
- 每个聊天会话会记住自己的项目目录。
- 如果找不到主程序或虚拟环境，会回退到可用的默认路径。

## 常见问题

`F5` 没反应：
- 先确认当前模型支持该功能。
- 先确认已经通过 `Ctrl+O` 加载项目。

提示找不到主程序：
- 确认项目中存在约定的入口文件名，例如 `main.py`。

运行失败：
- 查看控制台输出。
- 检查项目依赖和虚拟环境是否完整。
